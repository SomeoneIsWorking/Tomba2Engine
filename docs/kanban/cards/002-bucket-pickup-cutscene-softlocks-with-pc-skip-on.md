ROOT CAUSE (FIXED): beh_prng_velocity_machine (FUN_80117658) had a wrong branch constant. Guest 0x80117be0 `beq v1,v0` compares node[5] against v0 = the VARIANT byte loaded at 0x80117ba0 (==1 on that path). The transcription read v0 as 3 — the value 0x80117bd4 loads in the DELAY SLOT of the other branch's `j 0x80117cc0`, which never executes there. So node[5] latched at 1, the FUN_8005308C wait never ran, the end-of-pickup script never started, cut-mode stayed 1. Same defect CLASS as the seesaw/pickup fix (a guest register test transcribed wrong).

PROVEN EFFECT: the machine now advances HANDOVER->AWAIT_PLAYER->AWAIT_SCRIPT and the cutscene script RUNS — PSXPORT_DEBUG=pickup shows the cursor walk 80148574 -> 8014857C -> 80148584 -> 80148594 -> 8014859C -> 801485A4. Before the fix it never started.
