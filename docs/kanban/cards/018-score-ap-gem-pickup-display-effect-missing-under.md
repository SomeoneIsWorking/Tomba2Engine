**2026-07-23:** FIXED 2026-07-23. Root cause was NOT the 0x800EC188/FUN_8003D23C suspect (falsified: 0x800EC188 is the owned 40-slot OBJECT TABLE, FUN_8003D23C IS owned as leaf_8003D23C, and neither is in the popup chain). The popup's digits are emitted by the unowned drawer FUN_80071DFC (item/count sibling FUN_80072308) through the shared UI FT4 leaf FUN_8007E1B8, whose ONE native tap only recorded inside the pause-menu scope - so on the field the leaf ran the gen body only and the packets died in the OT that pc_render no longer walks.
